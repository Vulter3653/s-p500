# Repository Writing Rules

Updated: 2026-07-20

## 기준 문서

작업자는 변경 전에 `AGENTS.md`, 이 문서, `docs/progress.md`, `CHANGELOG.md`, `docs/debug-log.md`를 확인한다. 이 문서들이 채팅 기록보다 우선하는 저장소의 작업 기준이다.

## 변경 이력 작성

- 모든 의미 있는 변경은 `CHANGELOG.md`에 기록한다.
- `Added`, `Changed`, `Fixed`, `Removed`, `Security`, `Docs`, `Data`, `Analysis` 중 적절한 분류를 사용한다.
- 기록 형식은 `- Label: Description. (agent-id)`로 통일한다.
- 최신 버전과 항목을 위쪽에 추가하고 기존 이력을 보존한다.

## 진행 상황 작성

`docs/progress.md`에는 다음 내용을 기록한다.

- 완료된 작업과 현재 저장소 상태
- 검증 명령과 결과
- 아직 수행하지 않은 테스트
- 알려진 제한과 다음 작업
- 커밋 또는 배포와 관련된 인수인계 정보

## 디버그 기록 작성

`docs/debug-log.md`에는 다음 내용을 재현 가능한 형태로 기록한다.

- 날짜와 문제 요약
- 관찰된 증상과 오류 메시지
- 재현 조건 또는 실행 명령
- 확인한 원인과 근거
- 적용한 수정
- 수정 후 검증 결과
- 해결 여부와 남은 위험

단순히 "수정 완료"라고 기록하지 않는다. 원인을 확인하지 못했다면 추정과 확인된 사실을 구분한다.

## 데이터와 분석 결과

- 원본 데이터, 정제 데이터 및 분석 결과의 경로를 구분한다.
- 생성 결과에는 가능한 경우 생성 스크립트와 실행 명령을 연결한다.
- 수치를 수동으로 수정하지 않고 재현 가능한 코드로 생성한다.
- 표본 제외, 변수 정의, 결측치 처리 및 모델 변경은 문서에 명시한다.
- 분석 결과를 과장하지 않고 데이터가 지지하는 범위에서만 해석한다.

## 기록 보존과 수정 방식

- 로그의 기존 항목을 삭제하거나 압축하지 않는다.
- 기록 수정은 필요한 위치에만 제한적으로 적용한다.
- 다른 작성자의 귀속 표시를 변경하지 않는다.
- 중복이나 오류를 수정해야 한다면 수정 이유도 변경 이력에 기록한다.
